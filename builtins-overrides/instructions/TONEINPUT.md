**Summary**
- Like `TINPUT`, but uses the “one input” mode (`OneInput = true`).

**Syntax**
- Same as `TINPUT`.

**Arguments**
- Same as `TINPUT`.

**Defaults / optional arguments**
- Same as `TINPUT`.

**Semantics**
- Same as `TINPUT`, but the UI may truncate the entered text to at most one character (implementation detail).

**Errors & validation**
- Same as `TINPUT`.

**Examples**
- `TONEINPUT 5000, 0`
