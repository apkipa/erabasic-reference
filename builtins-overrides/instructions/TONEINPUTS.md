**Summary**
- Like `TINPUTS`, but uses the “one input” mode (`OneInput = true`).

**Syntax**
- Same as `TINPUTS`.

**Arguments**
- Same as `TINPUTS`.

**Defaults / optional arguments**
- Same as `TINPUTS`.

**Semantics**
- Same as `TINPUTS`, but the UI may truncate the entered text to at most one character (implementation detail).

**Errors & validation**
- Same as `TINPUTS`.

**Examples**
- `TONEINPUTS 5000, "A"`

**Progress state**
- complete
